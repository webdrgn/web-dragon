Playunmute

Играть

Я сделал браузерный тренажер для подготовки к техническим собеседованиям. Полный цикл: калибровка, интервью в Zoom-стиле, разбор ошибок с баллами и комментариями.

Зачем? Я провел десятки технических собеседований. Видел, как кандидаты готовятся не к тому. Сделал инструмент, который дает именно ту обратную связь, которая нужна.

  • Калибровка темпа мышления
  • Оценка содержания и скорости реакции
  • Разбор по шести этапам с вердиктом
  • Архитектура под новые сценарии без изменения ядра

Fullstack · Nuxt 3 · Vue 3 · TypeScript · Node.js · Express

Кейсы

Три проекта. Три результата.

На платформе витрина, кабинеты пользователей и турнирные таблицы жили отдельно друг от друга. Каждый новый формат события требовал ручной сборки экранов с нуля. Стили разъезжались, код дублировался, разработка тормозила. Я собрал библиотеку компонентов, настроил Storybook, договорился с дизайнерами. Теперь сборка — как конструктор. Типовые сценарии ускорились на 70%.

Проект, который боялись трогать. Мы мигрировали на Vue 3. Поэтапно, без остановки. Единые правила ревью, предсказуемые релизы. Десять проектов перевели. Команда перестала тратить двадцать часов в неделю на дубли и разбор легаси.

Проект рос, а вместе с ним падала скорость загрузки. Каждый релиз приносил новые баги, команда тратила время на исправления вместо новых фич. Мы договорились о целевых показателях производительности — выбрали метрики, за которыми следим в каждом релизе. Разобрались с узкими местами в загрузке страниц, пересобрали бандл. Параллельно пересмотрели подход к ревью на критических участках — проверять стали тщательнее. Результат: Core Web Vitals выросли до 95+, критических багов стало на 45% меньше.
